## Key architecture

### Layout
- **Desktop**: Header (tabs) ‚Üí Photo banner ‚Üí 3-column (LeftPanel | SharkMap | RightPanel) ‚Üí BottomStrip
- **Mobile**: Header ‚Üí Photo banner ‚Üí tabbed content (map/sharks/feed/research) ‚Üí bottom nav
- `App.jsx` owns all state and orchestrates layouts

### BottomStrip (`src/components/layout/BottomStrip.jsx`)
- Desktop (`forceOpen=false`): receives `activeTab` + `onTabChange` from App/Header, 350px fixed height
- Mobile (`forceOpen=true`): internal tab state, fills parent via `flex-1 min-h-0`
- Tab id `'flashcards'` renders `SharkFlashcards` ‚Äî labelled "Daily Quiz" in the UI

### Daily Quiz (`src/components/research/SharkFlashcards.jsx`)
- 5 True/False questions generated by Claude, cached daily
- Score stored in `sessionStorage` (key: `shark_flashcard_score_YYYY-MM-DD`)
- Completed score stored in `localStorage` (`shark_quiz_last_score`) for popup reference
- **Important**: `const total = cards.length` must be declared BEFORE the useEffect that references it in deps ‚Äî TDZ error otherwise

### Quiz welcome popup (`src/App.jsx` ‚Äî `QuizPopup` component)
- Shows once per day (gated by `shark_quiz_popup_date` in localStorage)
- Uses "Question 1" framing ‚Äî intentionally avoids "quiz" language
- On accept: sets `researchTab='flashcards'` and (mobile) `mobileTab='research'`

### Claude (`src/utils/gemini.js`)
- Model: `claude-haiku-4-5`
- In-memory session cache for all calls
- Daily content (quiz, history) tries `/daily/{type}-{date}.json` static file first, falls back to live API
- Pre-generation script: `~/sharks-cron/generate-daily.js` on MacBook Pro (runs 2am via launchd)

## Rules
- No backend ‚Äî client-side only
- Tab IDs are stable (`'flashcards'`, `'research'`, etc.) ‚Äî labels can change but IDs are used in state/localStorage
- `daily/` folder on the server is never wiped by deploy (rsync excludes it)
- Don't change Claude JSON schemas without updating the pre-generation script too
